## Как использовать (x32dbg)

### 1. Анализ бинарника

- Открываешь **BinNexus.Probe**
- Загружаешь DLL / EXE
- Запускаешь анализ
- Смотришь кандидатов

> [!TIP]
> Обращай внимание на высокий score и нужные API (network, memory, loader)

---

### 2. Открываешь в x32dbg

- Запускаешь x32dbg
- Открываешь файл или attach к процессу

---

### 3. Переход к функции

Берёшь **VA (адрес)** из BinNexus.Probe:

- Нажимаешь `Ctrl + G`
- Вставляешь адрес:

```text
0x6CED3D2
```

- Переходишь к функции

---

### 4. Анализ поведения

Смотришь:

- Вызовы API (`send`, `recv`, `LoadLibrary`)
- Логику (if, loop)
- Работа с памятью

> [!IMPORTANT]
> BinNexus.Probe даёт **точки входа**, а не полный анализ

---

### 5. Патчинг

Обычно:

- NOP инструкции
- Изменение переходов
- Подмена вызовов

Примеры:

- отключить условие
- изменить return
- убрать вызов функции

---

### 6. Проверка

- Запускаешь
- Смотришь результат
- Повторяешь при необходимости

---

## Итоговый процесс

```text
Probe → Найти → Понять → Изменить
```

```

> [!TIP]
> Используй BinNexus.Probe чтобы не реверсить “вслепую”
```
